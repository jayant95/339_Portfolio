nomadBookings()

The objective of this project was to propose and create a working prototype of a useful application for the Android OS. It had to incorporate several classes including intents, device hardware and a database. Group of 2

Research

We knew from the beginning that we did not want to create a game application. Instead, we wanted to create a utility application to push ourselves out of our comfort zones in order to learn about new technologies such as Google API's. The idea for a travel app came from my partner as she travels a lot and uses apps such as Expedia to look for flights. We thought this was a great idea for an app and wanted to expand so we could search for hotels, car rentals, and local attractions. We wanted this to be an all in one app aimed towards users who are constantly travelling.

Process

We created a timeline for each week so we would keep on track and work in a timely manner. We stuck with this throughout the development of the application. For our first milestone we implemented each tab that linked to each other in addition to explicit intents that took the user information and displayed the results in google. For our second milestone, a GPS feature was implement, which found the latitude and longitude and converted it into a city name. We planned to use this for the flights section of our app. However, when we learned of the QPX Express API, we realized that it needed an airport code so we instead moved the GPS location to the hotels, car rentals and attractions. This was so the user did not have to type in their location but instead was already there when they open the tab.

The milstone screens for Nomad Bookings. Main screen, car rentals, flights, and hotels. (Left to right)

By the final milestone the app was redesigned using calendars, dropdowns, buttons, and a design that looked more professional. The core idea of the app remained untouched but the UI of it was completely changed. QPX Express API was used to get valuable flight information regarding the users input. The user could input their location and destination from a preset dropdown list and departure date from a calendar which in turn would provide the price, airline, and duration for the top 3 deals. Since this data retrieval takes a few seconds I put it in a AsyncTask(separate thread) so the UI would not freeze. A loading spinner was added so the user knows that the app is loading the results. The database feature was accomplished with an SQLite Database to allow the user to save their search and allow to search for it later on.

The final product for Nomad Bookings. The revamped UI and the steps taken to find the cheapest flights available.

Challenges

The biggest challenge of this project was the implementation of the Google API QPX Express. This proved to be quite challenging as it was my first time using an API but was incredibly rewarding once working and implemented. There were not any examples showing how to even begin integrating QPX Express in Android Studio as most of them were done in PHP or C# for websites. This gave us a good starting point as we worked from there to get the data we wanted inside the app. This whole process took around a week to complete but was extremely necessary to get done to make the application practical and useful.

Outcome & Insights

We were quite satisfied with the outcome of this application. I am glad that we pushed ourselves to learn new technology and how Google API's work. We presented our application prototype successfully to a class of 30 including the teaching assistant and professor.
